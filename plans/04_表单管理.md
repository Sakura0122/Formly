# 表单管理

## 当前状态

### 已完成
- 已完成表单管理页主流程，覆盖目录树、本地搜索、分组/表单创建、重命名、表单移动、表单删除、顶部详情展示和预览空态。
- 已完成首页路由调整，`/` 默认进入管理页，管理页可携带 `formId` 跳转到编辑器。
- 已完成请求层复用与管理页数据封装，新增 `useRequest`、`form-group` / `form-definition` API 模块，页面不再重复维护请求 loading。
- 已完成后端最小补齐：支持根分组创建，表单详情返回补充 `createdBy`。
- 已完成管理页结构优化：创建/重命名/移动逻辑已下沉到弹窗组件，页面层 `useRequest` 已收敛为解构式调用。
- 已完成左侧目录树切换为 `el-tree + 自定义节点内容`，展开、当前选中与过滤已改由组件能力承接。
- 已完成左侧目录区域滚动容器切换为 `el-scrollbar`，并清理树节点组件无效 `expanded` 入参。
- 已完成删除链路切换为勾选式二次确认弹窗，并补齐分组删除入口。
- 已完成创建/重命名/移动类弹窗收敛为 `el-form + rules + validate`，并移除 footer 冗余包裹层。
- 已通过 `npm run type-check` 与 `npm run build` 验证前端构建链路。
