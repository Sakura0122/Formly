# 表单编辑器

## 当前状态

### 已完成
- 已完成编辑器三栏骨架、单主表格核心结构编辑、常用字段配置联动，以及选区/快捷键/复制粘贴/撤销恢复等基础交互。
- 已将核心状态与命令统一收口到 `Pinia editor store`，并完成 `EditorHeader / EditorCanvas / EditorConfigPanel` 的职责下沉。
- 已完成 schema 初始化加载、保存草稿与“保存并发布”首版闭环，头部状态已接入草稿 / 发布基础状态表达。
- 已处理历史快照与纯数据更新链路，当前 `table`、历史栈与剪贴板均已改为浅响应式容器。
- 已同步收口保存提示表达，普通保存改为“草稿已保存”，不再把每次保存直接表达成新增版本。
- 已切到“草稿 + 发布版本”新状态模型：移除 `currentVersionId` 前端依赖，统一按草稿保存与发布指针表达状态。
- 已同步完成头部与管理页状态收口：保存只落草稿、发布后切回已发布态，管理页版本展示仅依赖发布版本指针。
- 已完成右上角草稿预览：仅在存在已保存草稿时可点击，新开纯白窗口展示可填写预览，不读取未保存内存态。
- 已抽离共享表单预览渲染层，支持 `readonly / interactive` 两种模式，并与管理页发布预览复用。
- 已完成编辑器读取接口收口：设计页只消费单个草稿 `schema`，预览渲染统一直接消费该表格数据。
- 已补齐“已保存草稿存在态”判断，右上角预览不再依赖 `hasUnpublishedDraft`，而是按是否存在已保存草稿控制。
- 已删除未使用的 `schemaVersion` 字段，当前 schema 仅保留 `table` 作为持久化载荷。
- 已修正多选单元格的组件投放行为：组件库点击与拖拽命中选区时会按选区批量追加，并保持多选焦点与配置面板联动一致。
- 已调整右键菜单删行删列语义：文案改为“删除行 / 删除列”，并支持按当前选区批量删除涉及的行列。

### 未完成
- 导入、导出能力仍未接入。
- 数据源配置、字段映射、联动逻辑与表格级配置面板仍未开始。
- 多表并存、更稳定的持久化模型与更细粒度的会话恢复策略仍待明确。

### 当前问题
- 当前编辑器初始化接口的 `schema` 实际承载的是裸 `table` 数据，前端若继续按 `schema.table` 消费会造成展示错误。
- 重点缺口仍在更完整的批量结构操作、表格级配置和数据配置能力。
- 行高仍以估算值为主，多组件堆叠时仍可能出现宽度挤压与高度不稳。
- 当前已接入最小持久化闭环，但撤销/重做与“已保存快照一致”的细粒度脏状态识别仍未继续收敛。

## 实施计划
- 优先收口编辑器 `schema` 契约：前后端统一按裸 `table` 数据传输与初始化，移除多余包装层。
- 优先补齐导入、导出与更完整的 schema 能力，继续完善保存后的状态表达。
- 继续补齐表格结构能力，优先处理批量结构操作与表格级快捷编辑。
- 扩展右侧配置面板，增加表格级布局配置、数据源配置、字段映射与联动能力。
- 继续优化画布体验，重点处理高度估算、多组件堆叠和视觉反馈一致性。
- 继续评估页面离开时的 `editor store` 清理策略，为后续多表场景预留状态边界。
