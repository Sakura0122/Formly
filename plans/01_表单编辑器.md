# 表单编辑器

## 当前状态

### 已完成
- 已完成编辑器主流程：三栏骨架、单主表格编辑、字段配置、选区、多选投放、批量删行删列、删除组件、快捷键、复制粘贴与撤销恢复已闭环。
- 已将核心状态与命令收口到 `Pinia editor store`，并完成 `EditorHeader / EditorCanvas / EditorConfigPanel / EditorHistoryPanel` 的职责下沉；`table`、历史栈与剪贴板已收敛为纯数据链路。
- 已完成草稿/发布链路：初始化、保存草稿、保存并发布、头部状态、版本表达、已保存草稿判断与最小 schema 载荷均已统一，当前仅按裸 `table` 数据传输。
- 已完成预览链路：编辑器草稿预览与管理页发布预览复用同一套渲染层，支持 `readonly / interactive`，预览窗口已改为复用命名窗口并避免误报拦截。
- 已完成历史版本模式：支持列表浏览、只读预览、显式还原、未保存隔离与时间字段收口，右侧历史预览直接复用画布只读模式。
- 已收口画布容器与右键边界：自定义右键只作用于真实表格区域，空白区恢复浏览器原生菜单。
- 已完成粘贴导入首版：空白画布支持复制 Word / Excel 表格后直接粘贴，前端已接入空态提示、剪贴板监听、粘贴事件顺序修正与 schema 回填，后端已补齐 HTML / 纯文本表格解析、合并单元格映射，并将解析逻辑收口到独立工具类，已补齐编号式方法注释与本地静态检查修正。

### 未完成
- 导出能力仍未接入。
- 数据源配置、字段映射、联动逻辑与表格级配置面板仍未开始。
- 多表并存、更稳定的持久化模型与更细粒度的会话恢复策略仍待明确。

### 当前问题
- 行高仍以估算值为主，多组件堆叠时仍可能出现宽度挤压与高度不稳。
- 当前已接入最小持久化闭环，但撤销/重做与“已保存快照一致”的细粒度脏状态识别仍未继续收敛。
- 后端本地自动编译验证仍受环境约束：仓库缺少 `mvn`，替代性 `javac` 校验又被本机 Lombok / JDK 组合阻断，接口链路已完成但仍需在标准 Maven 环境补一次编译确认。

## 实施计划
- 继续补齐导出与更完整的 schema 能力，完善导入后的保存、发布与后续编辑体验联动。
- 收口后端粘贴导入解析实现，补齐独立工具类，降低 `FormDefinitionServiceImpl` 业务混杂度。

- 在标准 Maven 环境补跑后端编译 / 测试，确认 `paste-parse` 与现有 DTO / VO、Swagger 和序列化链路一致。

- 继续优化画布体验，重点处理高度估算、多组件堆叠与视觉反馈一致性。
