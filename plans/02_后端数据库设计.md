# 后端数据库设计

## 当前状态

### 已完成
- 已明确数据库阶段目标为支撑管理页、表单编辑器持久化和后续数据管理页。
- 已确认使用 MySQL，并采用完整历史版本、整份 schema JSON 落库的持久化方向。
- 已确认需预留提交数据存储，且提交记录绑定发布版本以保证历史可追溯。
- 已确认管理结构需同时支持无限级分组与根级表单。
- 已完成首版初始化脚本，已覆盖分组、表单、版本、提交记录四类核心表，以及常用索引、唯一约束和示例数据。
- 已收敛表单本体上的 `current_version_id` 与 `published_version_id` 指针，用于区分最新编辑版本和当前发布版本。
